# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in DeployKit, **please do not open a public issue**.

Instead, report it privately by emailing **cosmoframeworks@gmail.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You will receive an acknowledgment within **48 hours** and a detailed response within **5 business days** with next steps.

## Disclosure Policy

- We will work with you to understand and resolve the issue before any public disclosure.
- We will credit you in the release notes (unless you prefer to remain anonymous).
- We ask that you give us reasonable time to address the issue before disclosing it publicly.

## Scope

The following are in scope:

- The DeployKit application code (API, web dashboard, shared packages)
- The install/update/uninstall scripts
- Docker and Traefik configurations
- Authentication and encryption implementations

The following are out of scope:

- Vulnerabilities in third-party dependencies (report these upstream, but let us know)
- Issues that require physical access to the server
- Social engineering attacks
